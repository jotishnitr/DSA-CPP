## 🛠️ Problem-Solving Approach

Every problem in this repository is solved in **3 layers:**

```
Layer 1 → Brute Force     (understand the problem)
Layer 2 → Better Approach (optimize with data structures)
Layer 3 → Optimal         (best possible time/space)
```

Each file contains:
- ✅ Problem description with example
- ✅ Approach explanation (why it works)
- ✅ Step-by-step algorithm
- ✅ Time & Space complexity analysis
- ✅ Edge cases handled

## 💡 Tips for Fellow DSA Learners

> These are lessons learned from solving these 33 problems.

1. **Never skip Brute Force.** It builds intuition. Jumping to optimal directly without understanding brute force is a trap.

2. **Draw it out.** Two Pointers, Sliding Window, Matrix problems — always draw the state on paper before coding.

3. **Always ask: can I reduce an extra loop with a data structure?** A nested loop → O(n²) can often become O(n) with a HashMap.

4. **Memorize these 5 algorithms cold:** Kadane's, Moore's Voting, Dutch National Flag, Next Permutation, Binary Search. They appear constantly in interviews.

5. **`long long` saves you.** Whenever you multiply or sum large numbers (4Sum, Reverse Pairs, Missing+Repeating), cast to `long long` first.

6. **Practice recognizing patterns, not memorizing solutions.** When you see "sorted array + find pair" → Two Pointers. When you see "max subarray" → Kadane's. Build that reflex.

7. **Solve each problem at least twice** — once to learn, once to code from scratch without looking.
